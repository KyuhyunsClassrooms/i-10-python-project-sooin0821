# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: 신제품 바이럴 효과 및 마케팅 점수 계산기

이 프로그램을 만든 이유: 신제품 마케팅 시 홍보 채널(SNS, 블로그, 유튜브 등)별로 소비자의 호응도를 정량적으로 분석하기 위해서이다. 조회수 대비 좋아요 수의 비율인 '참여율'을 계산하고 성과 등급을 매김으로써, 마케터가 어떤 채널에 예산을 더 집중해야 하는지 데이터 기반의 의사결정을 내릴 수 있도록 돕고자 이 프로그램을 개발했다.


---

## 2. 문제 분해 및 모델링

### 입력
분석하고자 하는 가상의 신제품 이름(문자열), 마케팅 데이터를 확인하고 싶은 홍보 채널 번호(정수형: 1~3)

### 처리
1. 사용자가 선택한 채널 번호를 리스트 인덱스(번호 - 1)로 변환하여 2차원 리스트에서 해당 채널의 조회수와 좋아요수 데이터를 가져온다.
2. 수식 (좋아요수 / 조회수) * 100을 이용해 참여율(%)을 계산한다.
3. if-elif-else 조건문을 사용하여 참여율이 8% 이상이면 '대박', 4% 이상이면 '보통', 그 미만이면 '노력 필요' 등급을 판정한다.

### 출력
분석 제품명, 선택 채널 이름, 누적 조회수 및 좋아요수, 계산된 참여율(%), 최종 마케팅 성과 등급과 한 줄 피드백

---

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름:campaign

한 행의 의미:하나의 홍보 채널에 대한 마케팅 성과 데이터 세트

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 | 홍보 채널의 이름 | "SNS홍보" 또는 "유튜브숏폼" |
| 1번 열 | 해당 채널의 누적 조회수 | 5000 |
| 2번 열 | 해당 채널의 누적 좋아요수 | 250 |

이 2차원 리스트를 사용한 이유: 여러 개의 홍보 채널이 존재하고, 각 채널마다 채널명, 조회수, 좋아요수라는 연관된 다수의 데이터를 함께 묶어서 통합 관리해야 하기 때문이다. 1차원 리스트를 여러 개 만드는 것보다 2차원 리스트 구조를 활용하는 것이 데이터를 행과 열로 구조화하여 접근하기에 훨씬 효율적이다.

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| show_intro | 프로그램 제목과 환영 안내 문구 출력 | 없음 | 없음 (화면 출력) |
| get_user_input | 신제품 이름과 원하는 채널 번호 입력받기 | 없음 | product_name, channel_num |
| calculate_and_evaluate | 데이터 인덱싱, 참여율 계산 및 등급 판정 | data, channel_num | channel_name, views, likes, engagement_rate, grade |
| print_result | 최종 마케팅 분석 보고서 형식으로 출력 | 제품명, 채널명, 조회수, 좋아요수, 참여율, 등급 | 없음 (화면 출력) |

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름: calculate_and_evaluate

설명: 이 함수는 프로그램의 핵심 로직(처리)을 담당한다. 메인 함수로부터 2차원 리스트 데이터와 사용자가 고른 번호를 입력받아, 리스트 인덱스에 맞게 행을 선택한다. 그 후 특정 열에 저장된 조회수와 좋아요수를 추출하여 참여율 공식을 통해 연산하고, 조건문으로 마케팅 성공 등급을 분류하여 최종 결과를 반환하는 핵심적인 데이터 분석 역할을 수행한다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

어떤 조건을 검사했는가? calculate_and_evaluate 함수 내부에서 계산된 참여율(engagement_rate) 변수의 값을 검사했다. 값이 8 이상인지, 4 이상인지 비교하여 각각 다른 등급 문자열을 변수에 할당하도록 구현했다.


### 반복문 사용 부분

무엇을 반복 처리했는가? main 함수나 안내 함수에서 2차원 리스트 campaign을 순회하며 현재 등록되어 있는 홍보 채널들의 이름(0번 열)을 하나씩 꺼내어 화면에 목록으로 출력할 때 반복문을 활용할 수 있다.

### 조건문과 반복문이 함께 작동한 부분
보고서의 '조건문과 반복문 결합 구조' 항목을 채우기 위해, for문 내부에서 if문으로 성과를 판별하는 show_all_summary 함수를 새로 정의하고 main 함수에 결합하여 프로그램의 데이터 분석 기능을 고도화했다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력:제품명 "텀블러", 채널 번호 3

예상 결과:: 유튜브숏폼 데이터 분석, 참여율 10.00%, '대박' 등급 출력

실제 결과:예상한 대로 소수점 둘째 자리까지 정확히 표현되며 보고서가 출력됨.


### 테스트 2: 예외 또는 경계 사례

입력:채널 번호에 1~3 이외의 숫자나 문자를 넣는 경우

예상 결과:리스트의 범위를 벗어나 IndexError 등이 발생하거나 프로그램이 멈출 수 있음.

실제 결과:(현재 코드 기준) 인덱스 범위를 초과하는 에러가 발생함.


### 발견한 오류와 수정한 방법
처음 코드를 설계할 때 리스트의 인덱스가 0부터 시작한다는 점을 간과하여, 사용자가 입력한 번호 그대로 data[channel_num]과 같이 접근했을 때 잘못된 행을 가져오는 오류가 있었다. 이를 해결하기 위해 사용자가 입력한 직관적인 번호(1, 2, 3)에서 1을 빼서 인덱스 규칙(idx = channel_num - 1)을 맞추도록 수정하여 해결했다.

## 8. Gemini 활용 내용 요약

Gemini에게 도움받은 부분:아이디어가 구체적이지 않을 때 진로(마케팅)와 연계된 2차원 리스트 활용 주제들을 추천받았고, 입력-처리-출력의 흐름에 맞게 함수를 분리하는 전체적인 코드 구조(스켈레톤 코드)를 설계하는 데 도움을 받았다.


Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분: Gemini는 처음 주제 추천 시 calculate_engagement와 evaluate_success라는 2개의 함수로 완전히 쪼개는 방식을 제안했으나, 코드의 복잡도를 낮추고 인덱스 접근을 한 번에 처리하기 위해 두 기능을 calculate_and_evaluate라는 하나의 핵심 처리 함수로 병합하여 수정 적용했다.


최종 코드에서 내가 직접 판단하거나 수정한 부분: 결과 출력 함수(print_result)에서 마케팅 보고서처럼 보일 수 있도록 대시(-)와 선(=)을 활용해 가독성을 높였고, 참여율이 소수점으로 지저분하게 나오지 않도록 파이썬의 f-string 포맷팅 문법(:.2f)을 직접 조사하여 반영했다.

## 9. 한계와 개선 방안

현재 프로그램의 한계:사용자가 1~3번 이외의 잘못된 번호나 문자를 입력했을 때 에러가 나면서 프로그램이 강제 종료되는 한계가 있으며, 2차원 리스트에 새로운 홍보 채널 데이터를 사용자가 직접 추가하는 기능이 없다.


다시 개선한다면 추가하고 싶은 기능:try-except 문이나 while 반복문을 사용해 잘못된 입력이 들어와도 에러 없이 "다시 입력하세요"라고 안내하는 예외 처리 기능을 추가하고 싶다. 또한, 사용자가 직접 새로운 채널과 데이터를 입력해 리스트에 추가(append)할 수 있는 '채널 등록 기능'을 구현해보고 싶다.

## 10. 코드인터뷰 준비

다음 질문에 답할 수 있도록 준비하세요.

1. 내 프로그램의 입력-처리-출력은 무엇인가?
답변: "네, 제 프로그램의 입력은 분석할 가상의 신제품 이름과 사용자가 선택하는 홍보 채널 번호(1~3번)입니다.
처리는 선택된 채널의 조회수와 좋아요수 데이터를 2차원 리스트에서 가져와 '참여율'을 공식에 맞게 계산하고, if 조건문으로 마케팅 성공 등급을 매기는 과정입니다.
마지막으로 출력은 분석된 참여율과 성과 등급, 그리고 채널별 맞춤형 한 줄 피드백을 보고서 형태로 화면에 인쇄해 주는 것입니다."
2. 2차원 리스트에서 한 행과 각 열은 무엇을 의미하는가?
답변: "제 프로그램에서 사용한 2차원 리스트의 이름은 campaign입니다. 여기서 한 행은 하나의 홍보 채널에 대한 마케팅 성과 데이터 세트를 의미합니다.
각 열의 의미를 말씀드리면, 0번 열은 '홍보 채널의 이름(예: SNS홍보, 유튜브숏폼)', 1번 열은 '누적 조회수', 2번 열은 '누적 좋아요수'를 뜻합니다. 여러 채널의 연관된 데이터를 행과 열로 묶어서 관리하기 위해 2차원 리스트를 사용했습니다."
3. 가장 중요한 함수는 무엇이고 어떤 역할을 하는가?
답변: "가장 중요한 함수는 데이터 연산과 판정을 담당하는 calculate_and_evaluate 함수입니다.
메인 함수로부터 2차원 리스트 데이터와 사용자가 선택한 채널 번호를 넘겨받은 뒤, 사용자의 번호를 리스트 인덱스 규칙에 맞게 매칭하여 해당 채널의 조회수와 좋아요수를 정확히 추출합니다. 이를 통해 참여율을 계산하고 조건문으로 최종 등급까지 판정하여 반환해 주는 가장 핵심적인 분석 엔진 역할을 합니다."
4. 조건문과 반복문은 어디에서 사용되었는가?
답변: "우선 조건문은 calculate_and_evaluate 함수 안에서 계산된 참여율이 8% 이상이면 '대박', 4% 이상이면 '보통', 그 미만이면 '노력 필요' 등급을 나눌 때 if-elif-else 구조로 사용되었습니다.
반복문은 get_user_input 함수에서 현재 등록된 모든 채널 이름을 화면에 목록으로 자동 출력할 때 for문이 사용되었습니다.
특히 이 두 제어구조가 함께 작동한 부분은 show_all_summary 함수입니다. for문으로 모든 채널을 하나씩 순회하는 동시에, 그 내부에서 if문으로 참여율이 8% 이상인 우수 채널을 판별하여 앞에 별표(★)를 붙여 특별하게 출력하도록 결합하여 구현했습니다."
5. Gemini가 도와준 부분과 내가 직접 수정한 부분은 무엇인가?
답변: "Gemini가 도와준 부분은 제 진로인 마케팅 아이디어를 구체화하여 2차원 리스트의 데이터 구조를 설계해 준 것과, 프로그램을 입력-처리-출력 기능별로 깔끔하게 분리할 수 있도록 전체적인 함수의 뼈대를 잡아준 것입니다.
제가 직접 수정한 부분은 Gemini가 처음 제안했던 복잡한 함수 구조를 하나로 병합하여 효율적으로 바꾸었고, 보고서 양식에 맞게 선(=)과 대시(-)를 넣어 출력 화면을 보기 좋게 디자인했습니다. 또한, 참여율이 깔끔하게 출력되도록 소수점 둘째 자리 제한 문법(:.2f)을 직접 조사하여 반영했고, 필수 조건인 반복문 결합 구조를 만족하기 위해 전체 요약 리포트 함수(show_all_summary)를 직접 추가하여 코드를 고도화했습니다."
