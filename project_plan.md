# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 신제품 바이럴 효과 및 마케팅 점수 계산기

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

내 설명: 신제품을 홍보할 때 어떤 채널(SNS, 블로그, 유튜브 등)의 마케팅 효율이 좋은지 데이터 분석이 필요하다. 사용자가 확인하고 싶은 채널을 선택하면, 조회수와 좋아요수를 바탕으로 참여율을 자동 계산하고 성과 등급을 매겨주는 프로그램을 만들어 마케팅 의사결정을 돕고자 한다.

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 홍보를 진행할 가상의 신제품 이름 (문자열)
- 마케팅 데이터를 확인하고 싶은 홍보 채널 번호 (정수형: 1, 2, 3 중 선택)

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 2차원 리스트에서 사용자가 선택한 채널의 조회수와 좋아요수 데이터 가져오기
- 참여율 계산 공식: (좋아요수 / 조회수) * 100으로 계산하기
- 계산된 참여율에 따라 if-elif-else 조건문을 사용하여 마케팅 성공 등급(대박/보통/노력 필요) 나누기

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 선택한 채널의 마케팅 데이터 현황 (채널명, 조회수, 좋아요수)
- 계산된 참여율(%) 및 최종 마케팅 성과 등급
- 등급에 따른 마케팅 개선 방향 한 줄 피드백

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름: campaign
campaign = [
    ["SNS홍보", 5000, 250],      # 0번째 행: [채널명, 조회수, 좋아요수]
    ["블로그리뷰", 3000, 150],    # 1번째 행: [채널명, 조회수, 좋아요수]
    ["유튜브숏폼", 12000, 1200]   # 2번째 행: [채널명, 조회수, 좋아요수]
]

### 한 행의 의미
하나의 홍보 채널에 대한 마케팅 성과 데이터 세트

### 각 열의 의미

| 열 번호 | 의미 | 예시 |
|---:|---|---|
| 0번 열 | 홍보 채널의 이름 | "SNS홍보" |
| 1번 열 | 해당 채널의 누적 조회수 | 5000 |
| 2번 열 | 해당 채널의 누적 좋아요수 | 250 |
---

## 5. 함수 설계

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| show_channels | 현재 등록된 채널의 종류를 화면에 보여줌 | campaigns (2차원 리스트) | 화면 출력 (반환값 없음) |
| calculate_engagement | 조회수와 좋아요수로 참여율(%)을 계산함 | views (조회수), likes (좋아요수) | 계산된 참여율 (실수형 변수) |
| evaluate_success | 참여율을 기준으로 마케팅 등급을 판정함 | engagement_rate (참여율) | "대박", "보통", "노력 필요" 중 하나 (문자열) |

---

## 6. 사용할 제어구조

### 조건문을 사용할 부분

내 계획: 사용자가 입력한 참여율 점수가 8% 이상이면 '대박', 4% 이상이면 '보통', 그 미만이면 '노력 필요'를 판정할 때 if-elif-else 조건문을 사용한다. 또한, 사용자가 번호를 잘못 입력했을 때 예외를 처리하는 조건문에도 활용한다.


### 반복문을 사용할 부분

내 계획:프로그램 시작 시 현재 2차원 리스트(campaigns)에 들어있는 모든 홍보 채널의 이름과 데이터를 하나씩 화면에 보기 좋게 출력할 때 for 반복문을 사용한다.

## 7. 예상되는 예외 상황

1. 사용자가 채널 번호를 선택할 때 1, 2, 3번이 아닌 잘못된 숫자(예: 5번)나 문자(예: "a")를 입력하는 경우
2. (데이터 설정 오류로) 특정 채널의 조회수가 0으로 설정되어 있어, 나눗셈 계산 시 오류(ZeroDivisionError)가 발생하는 경우
